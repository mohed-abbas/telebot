<objective>
Close UAT Gap #1: the compiled `app.<hash>.css` shipped by Phase 5 Plan 02
contains **zero Basecoat rules and zero compat-shim rules**. Root cause: the
vendored `basecoat-css@0.3.3` is a **Tailwind-v4 library** (uses
`@custom-variant`, `@theme`, `@apply @container/*`, `has-data-[slot=...]`), but
Plan 02 pinned the **Tailwind v3.4.19 standalone CLI** — which cannot parse
v4 directives AND silently drops CSS `@import` statements. Result: the login
page renders bare HTML and the `alert-destructive` error banner renders as
plain text.

**Locked decision (D-04-REVISED in CONTEXT.md):** bump the Tailwind standalone
CLI to **v4.x** (still no Node runtime — v4 ships a Linux binary at
`github.com/tailwindlabs/tailwindcss/releases`). v4 natively resolves
`@import` and parses Basecoat's v4 directives. `_compat.css` stays as-is
(v4 supports `@apply`).

Purpose: make `/login` render as designed (styled Basecoat card + red
`alert-destructive` banner on wrong password) and give every future Phase 7
template the same substrate. Lock in a build-time regression test so the next
Basecoat bump (or Tailwind bump) cannot silently drop the vendored rules again.
