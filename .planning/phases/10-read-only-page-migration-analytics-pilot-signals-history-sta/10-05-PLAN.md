## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser SPA → /api/v2/signals, /api/v2/history | Authenticated GET via api() (session cookie) |
| URL search params → history query key | 5 filter values flow from URL → request |
| signals details/raw_text → DOM | Server-controlled-but-channel-sourced text rendered into the page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-11 | Tampering/Info (reflected XSS) | SignalsView Details cell | mitigate | Render `details`/`raw_text` as React text children; grep guard forbids dangerouslySetInnerHTML in SignalsView |
| T-10-12 | Tampering (SQLi) | history filter params | accept | Server parameterizes all 5 params (asyncpg $n, verified); SPA passes only the known keys |
| T-10-13 | Spoofing | both reads | mitigate | api() credentials same-origin httpOnly cookie; 401 → inherited global redirect |
| T-10-14 | Information Disclosure | rendered prices | mitigate | Money/price from server `_display` strings only (Pitfall 5) |
| T-10-SC | Tampering | shadcn CLI source-gen | accept | `npx shadcn add select` copies vetted source, not an npm dep (RESEARCH §Package Legitimacy) |
</threat_model>
