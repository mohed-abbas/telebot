## Trust Boundaries (financial-safety framing)

|----------|-------------|
| signal → persisted staged row | The signal's real SL/TP must survive the in-memory signal being garbage-collected; if it is lost, later stages fire with wrong protective levels |
| schema migration → live table | A no-op migration (Pitfall 1) leaves columns absent → inserts fail or read path silently loses SL/TP |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-13-02 | Information disclosure / loss of protective levels | create_staged_entries / get_active_stages | mitigate | Columns nullable + `.get()` on row dict so pre-migration rows are NULL-safe; Plan 02 read path falls back to default SL on NULL (never to sl=0) |
