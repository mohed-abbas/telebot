<objective>
Fix EXEC2-01 on the READ side (executor.py only). Two coupled effects of the same persisted-column mechanism (RESEARCH Pitfall 2 + D2-05):

1. **Late-stage SL/TP carry:** `_fire_zone_stage` currently rebuilds SL from `default_sl_pips` and sets `target_tp=None` (executor.py:799-820), so a stage fired hours later by the watchdog loses the signal's real protective levels. Source `signal_sl` / `signal_tp` from the persisted stage row (added by Plan 01) instead.

2. **Revive the dormant price-cascade:** The cascade at executor.py:585-639 reads `get_signal_targets(signal_id)`, but for correlated sequences `signal_id` is the orphan (sl=0, tp=0), so it is a SILENT NO-OP — deep stages keep firing after price already hit the real TP. Re-source the cascade's SL/TP from the persisted per-stage columns so it actually fires.

Both must be NULL-safe: a pre-migration row with NULL `signal_sl` falls back to the existing `default_sl_pips` SL behavior — NEVER sl=0 (D-08 backstop preserved).

Purpose: Every stage of a sequence ends with consistent, signal-derived SL/TP, and the cascade protects deep stages.
Output: executor.py read path uses persisted SL/TP.
</objective>
