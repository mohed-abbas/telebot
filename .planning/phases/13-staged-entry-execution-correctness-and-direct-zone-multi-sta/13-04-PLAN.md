<objective>
Implement EXEC2-05 (D2-09..D2-12): an orphan text-only position — stage 1 filled at market with default SL, no follow-up in the correlation window — must never be left unmanaged. At window expiry, attach a protective TP so the position rides on its default SL + an assigned TP.

All locked constraints:
- **D2-10/D2-11:** protective `TP = entry ± (sl_distance × R)`, **R = 1:1 hardcoded** (TP distance == SL distance). No new config knob. `sl_distance = default_sl_pips × pip_size` (the same value used at orphan stage-1 open).
- **D2-12:** Attach at window-expiry ONLY — during the window, leave the position on default SL with NO TP (a fast follow-up still gets to set the real SL/TP first). No new task/loop — reuse the existing 10s `_zone_watch_loop` as the watchdog. Reuse the stage-1-align `modify_position` pattern (failure-isolated).

**Resolved open questions (RESEARCH Open Q1/Q2):**
- *Window-expiry detection:* the in-memory correlator pops the orphan on pairing, so it cannot be queried at expiry. Detect via the DB: a `stage_number=1` row whose signal is older than `correlation_window_seconds`, with a live `mt5_ticket`, AND **no sibling stage rows** (no stage_number≥2 for that signal_id — a follow-up would have created siblings).
- *Idempotency:* check the live position's CURRENT TP before setting — skip if already non-zero. This needs NO schema change (preferred over a persisted flag). Must survive reconnect/loop re-ticks.

This plan is **autonomous: false** — it carries a human-verify checkpoint for the live-money round-trip (DryRunConnector cannot exercise the real MT5 modify/TP; deploy-at-end per project_deploy_at_end_workflow.md). The automated DryRun tests gate the logic; the checkpoint gates the live behavior at final VPS acceptance.

Purpose: no unmanaged orphans; supersedes Phase 6 D-09.
Output: executor.py orphan protective-TP attach in the existing loop.
