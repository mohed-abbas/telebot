<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/v2/auth/me (boot guard) | session validity decided server-side; SPA only reacts to 200/401 |
| browser → /api/v2/auth/logout | state-changing POST requiring the CSRF header |
| browser → /api/v2/trading-status (probe poll) | read-only in-memory flag; auth-gated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-10 | Elevation of Privilege | shell render before auth | mitigate | Boot guard gates the shell on `GET /auth/me`; every data call is independently auth-gated server-side (require_user → 401). No client-trusted auth state. |
| T-09-11 | Denial of Service | 401 redirect loop from boot guard | mitigate | The boot guard delegates the 401 redirect to the single global onAuthError (loop-break on /app/login); App.tsx adds no competing redirect (single bounce, SPA-04). |
| T-09-12 | Tampering | logout without CSRF | mitigate | Sign out POSTs through the http wrapper which sets X-CSRF-Token from the readable cookie (Phase 8 logout requires it). |
| T-09-13 | Information Disclosure | probe polling leaks state | accept | `/trading-status` returns only a boolean paused flag, auth-gated; no PII, negligible value. Single-operator internal tool. |
</threat_model>
