CSRF double-submit (api/deps.py):
  CSRF_COOKIE = "telebot_csrf"   (httponly=False, samesite="lax", path="/" ŌĆö READABLE by JS)
  CSRF_HEADER = "x-csrf-token"
  state-changing methods = POST, PUT, PATCH, DELETE  ŌåÆ server compares cookie vs X-CSRF-Token header
    (secrets.compare_digest); mismatch/absent ŌåÆ 403. GET is exempt.
Session cookie: telebot_session  (httpOnly ŌĆö set by SessionMiddleware; NEVER read in JS).

Auth endpoints (api/auth.py ŌĆö all under /api/v2):
  GET  /api/v2/auth/csrf   ŌåÆ 200 {"csrf_token":"<t>"} + Set-Cookie telebot_csrf=<t> (readable). Body token == cookie value.
  GET  /api/v2/auth/me     ŌåÆ 200 {"user":"admin"} | 401 {"error":{"code":"unauthorized","message":"Session expired"}}
  POST /api/v2/auth/login  ŌåÆ body {"password":"...","csrf_token":"<echo of telebot_csrf cookie>"} + X-CSRF-Token header
                             200 {"user":"admin"} + Set-Cookie telebot_session (+ refreshed telebot_csrf)
                             401 {"error":{"code":"unauthorized","message":"invalid_credentials"}}
                             403 {"error":{"code":"forbidden","message":"CSRF token invalid"}}   ŌåÉ cold-start (Pitfall 5)
                             429 {"error":{"code":"rate_limited","message":"rate_limited"}}
  POST /api/v2/auth/logout ŌåÆ requires X-CSRF-Token header; 200 {"ok":true}

Error envelope (api/errors.py): failures = {"error":{"code","message","fields"?}}; success bodies are BARE.
  statusŌåÆcode: 401ŌåÆunauthorized, 403ŌåÆforbidden, 422ŌåÆvalidation_error, 429ŌåÆrate_limited, 503ŌåÆunavailable.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser JS ŌåÆ /api/v2 (cookie auth) | the SPA submits credentials via the httpOnly session cookie + readable CSRF cookie |
| login form ŌåÆ /api/v2/auth/login | untrusted password input crosses to the server (argon2 verify server-side, Phase 8) |
| dev: Vite origin ŌåÆ dev container | cross-port risk mitigated by the same-origin /api proxy (Plan 01) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Information Disclosure | auth token storage | mitigate | httpOnly `telebot_session` never read in JS; nothing in localStorage/sessionStorage (grep-enforced); `credentials:"same-origin"` only. (SPA-03) |
| T-09-07 | Tampering | CSRF on mutations | mitigate | http.ts echoes the readable `telebot_csrf` cookie as `X-CSRF-Token` on POST/PUT/PATCH/DELETE; cross-site can't set the custom header; server compares with compare_digest (Phase 8 D-15). |
| T-09-08 | Denial of Service | 401 redirect loop | mitigate | `onAuthError` skips the redirect when already on `/app/login`; hard-nav clears in-memory state (single bounce, SPA-04). |
| T-09-09 | Information Disclosure | dev cross-origin cookie leak | mitigate | Relative `/api/v2` URLs only (never absolute :8090); same-origin Vite proxy; never credentialed-wildcard CORS, never weaken cookie attrs (Pitfall 4). |
</threat_model>
