## Spacing Scale

8-point scale (Tailwind v4 default 0.25rem step; use the named scale — all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, label→input gap |
| md | 16px | Default element spacing, card padding, form field gap |
| lg | 24px | Section padding, login card inner padding |
| xl | 32px | Layout gaps, main content area padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- **Interactive touch targets** (nav links, mobile menu button, icon buttons): **min 40px**
  height — matches the legacy sidebar row height (`btn-icon-ghost` / `sidebar-link`). Keep
  ≥40px so the shell reads identically to legacy and stays tap-friendly on the mobile drawer.
- **Sidebar fixed width: 224px** on desktop (`14rem` / Tailwind `w-56`; main content offset
  `md:ml-56`) — matches the legacy `md:ml-56` for visual continuity during parallel-run.
  On mobile the sidebar collapses to a drawer (hidden by default, toggled by a top-bar button).

## Typography

**Font families (tokens):**
- `--font-sans` (body default): `system-ui, -apple-system, "Segoe UI", Roboto, Helvetica, Arial, sans-serif`
- `--font-mono` (numeric / price / PnL readouts — established now for later pages; Phase 9 has
  no numeric tables but the probe's status/timestamp readout uses it): `ui-monospace, "SF Mono", "Cascadia Code", Menlo, monospace`

**Type scale — exactly 4 sizes, exactly 2 weights:**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | Page / login title ("Telebot" login heading) |
| Heading | 20px | 600 (semibold) | 1.2 | Section headings (e.g. probe "Connection probe" heading) |
| Body | 16px | 400 (regular) | 1.5 | Default body text, input values, paragraph copy |
| Small / Meta | 14px | 400 regular (600 for active nav / CTA label) | 1.5 | Nav links, meta text, brand subtitle, field labels, button labels |

**Weight rule:** regular **400** for body/meta; semibold **600** reserved for headings,
the **active** nav-link label, and the primary CTA label. No other weights.

## Color

**Visual hierarchy / focal point (per-view):**
- **Login view** — primary visual anchor is the centered login card (`--card` `#1a1a2e` on the
  `#0f0f1a` field) with the "Telebot" title at Display 28px/600; the single cyan **"Log in"** CTA
  is the only saturated element, so it is the eye's terminus.
- **App shell** — the dominant focal point is the **sidebar `--primary` cyan wordmark + the active
  nav indicator** (the only saturated elements in the chrome); the content area is intentionally
  quiet (empty state or the throwaway probe). The probe widget, when present, is the single content
  focal point but is explicitly diagnostic/secondary — it must never out-weigh the nav.

Brand palette mapped to shadcn semantic roles (D-10), converted hex→oklch for Tailwind v4
consistency. 60/30/10 split:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f1a` → `--background` ≈ `oklch(0.12 0.02 275)` | App background, main content surface |
| Secondary (30%) | `#1a1a2e` → `--card` / `--popover` / `--sidebar` ≈ `oklch(0.17 0.03 275)` | Cards, login card, sidebar, popovers, nav surface |
| Elevated / muted | `#252542` → `--muted` / `--border` / `--input` ≈ `oklch(0.24 0.04 275)` | Muted surfaces, borders, input borders, disabled-nav background |
| **Accent (10%)** | **`oklch(0.7 0.13 200)` (cyan/teal)** → `--primary` / `--ring` / `--sidebar-primary` | **See "Accent reserved for" — exclusive list only** |
| Accent foreground | near-black ≈ `oklch(0.18 0.02 275)` → `--primary-foreground` | Text/icon on the cyan accent fill (contrast on the cyan button) |
| Foreground | near-white ≈ `oklch(0.97 0 0)` → `--foreground` | Body text on `#0f0f1a` (WCAG AA) |
| Muted foreground | mid-grey (shadcn derived) → `--muted-foreground` | Disabled nav labels, brand subtitle, meta text |
| Destructive | red ≈ `oklch(0.62 0.21 25)` → `--destructive` | **Themed now, UNUSED in Phase 9.** Present so Phase 11 (D-10) live-money buttons inherit it. No Phase-9 control uses it. |

**Accent (cyan `--primary`) reserved EXCLUSIVELY for:**
1. The login submit CTA ("Log in" button fill)
2. The **active** nav-link indicator (active route in the sidebar)
3. Focus rings (`--ring`) on inputs / buttons / interactive elements
4. The brand wordmark "Telebot" in the sidebar header (text-primary, matching legacy)

> Never apply the cyan accent to "all interactive elements," disabled nav links, body text,
> or status indicators. It is a 10% accent — scarcity is the contract.

**Status semantics (independent of the cyan accent) — for the sidebar trading-state dot:**
- **green** = LIVE / trading enabled
- **red** = disabled
- **amber / muted** = DRY-RUN

These are STATUS signals, NOT the accent. Do NOT use cyan for status. (Phase 9 may render the
dot driven by the probe's `GET /api/v2/trading-status` `status` field; full status UI is later.)

**Verification:** render a Card/Input/Button and confirm opaque, correct dark brand colors
before building the shell (RESEARCH Pitfall 9).

## Copywriting Contract

Phase 9 scope = login + shell + diagnostic probe only. Mirrors the Phase-8
`/api/v2/auth/login` contract and the legacy nav labels for continuity.

### Login view

| Element | Copy |
|---------|------|
| Login title | **Telebot** (brand, display 28px) |
| Username field label | **Username** |
| Password field label | **Password** |
| Primary CTA (idle) | **Log in** (sentence case) |
| Primary CTA (pending / submitting) | **Logging in…** (button disabled while pending) |
| Error — 401 / bad credentials | **Incorrect username or password.** (inline; no field-specific leak) |
| Error — network / 500 | **Something went wrong. Please try again.** |
| Session-expired banner (optional, only when arrived via 401 redirect — derive from a query param/flag) | **Your session expired — please log in again.** (subtle banner above the form) |

> Cold-start CSRF note (RESEARCH Pitfall 5): the login view must `GET /api/v2/auth/csrf` on
> mount to seed the readable `telebot_csrf` cookie BEFORE the first `POST /auth/login`, or the
> first attempt 403s. This is a behavioral contract, not copy — but the executor must wire it.

### App shell (sidebar nav skeleton — D-07, disabled-visible)

| Element | Copy |
|---------|------|
| Brand wordmark | **Telebot** (text-primary cyan) |
| Brand subtitle | **Trading Dashboard** (muted-foreground) |
| Live nav link (the probe/landing route) | **Overview** |
| Disabled-visible future links (greyed, muted-foreground, NOT hidden) | **Positions**, **Trade History**, **Signal Log**, **Analytics**, **Pending Stages**, **Settings** |
| Sign-out link (sidebar footer) | **Sign out** |

> Decision (within D-07 discretion): future-page links render **disabled-visible** (greyed via
> `muted-foreground`, non-interactive), NOT hidden — so the shell reads complete and Phase 10
> slots real pages into ready-made routes. Labels match legacy exactly for parallel-run continuity.

### Shell content area (no real page yet)

| Element | Copy |
|---------|------|
| Empty state heading | **Select a section to get started.** |
| Empty state body | (neutral placeholder — or the throwaway probe widget occupies this area) |
| Probe widget heading | **Connection probe** (clearly labeled as a scaffold/diagnostic — removed in Phase 10) |
| Probe widget body | Live polling readout from `GET /api/v2/trading-status` (status + last-updated timestamp, mono) plus a deliberately-open `<input>` proving the SC#5 split — text survives ≥2 refetch cycles uncloberred. |

### Boot-guard / global-401 behavior (D-05 / D-06)

- On 401 (boot guard or mid-session) → **silent hard-nav** to `/app/login` (no toast).
- Loop-break: do NOT re-redirect when already on `/app/login`.
- The login view MAY show the subtle "Your session expired…" banner when arrived-at via 401.

### Destructive actions

**NONE in Phase 9.** No kill switch, no mutations, no confirmations. The `--destructive`
token is themed but unused. Destructive copy + confirmation flows are deferred to Phase 11.

| Element | Copy |
|---------|------|
| Destructive confirmation | not applicable — no destructive actions in this phase |
