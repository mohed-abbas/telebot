<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → uvicorn /app/* | untrusted deep-link path requests cross into StaticFiles serving |
| browser → uvicorn /api/v2/* | the API surface that must NOT be shadowed by the SPA mount |
| nginx → telebot:8080 | the single proxy hop; no new location block needed under D-02 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03 | Tampering | /app deep-link fallback | mitigate | Fallback is a StaticFiles subclass scoped to the /app mount, registered AFTER the /api/v2 router — structurally cannot return index.html for /api/v2/* (serving test `test_api_not_shadowed_by_spa_mount` proves it). |
| T-09-04 | Denial of Service | nginx routing during parallel-run | accept | No root `try_files` catch-all is added (Pitfall 6); existing `location /` proxy reaches uvicorn-served /app/. Low risk: single-operator internal tool. |
| T-09-05 | Spoofing | unauthenticated /api/v2 access via SPA | mitigate | `/app` serves only static assets; every data call still hits `require_user` → 401 (no auth bypass introduced; bot core/api untouched, diff-guard enforced). |

> nginx `/api/v2/auth/login` rate-limit block (Phase 8 D-14) is a DEPLOY artifact, not app code, and
> is currently absent (nginx/telebot.conf only rate-limits the legacy `= /login`). This is flagged
> for the deploy/UAT step, not gated in this plan (RESEARCH Assumption A4 / Pitfall 6).
</threat_model>
