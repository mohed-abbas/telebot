### `dashboard.py` (MODIFY — add `/app` static mount + deep-link fallback)

**Analog:** `dashboard.py:234` — the existing `/static` mount is the precise template for the new `/app` mount. Same `StaticFiles` import (`dashboard.py:23`), same `BASE_DIR` pattern.

**Existing mount pattern to copy** (`dashboard.py:234`):
```python
app.mount("/static", StaticFiles(directory=str(BASE_DIR / "static")), name="static")
```

**Router-registration ordering is load-bearing** (`dashboard.py:216-220`): the `/api/v2` router and error handlers are wired BEFORE any `app.mount`. The new SPA mount and its deep-link fallback MUST be added AFTER `app.include_router(api_router)` (line 219) and ideally after the existing `/static` mount (line 234), so `/api/v2/*` route handlers always win precedence over the `/app/*` catch-all (RESEARCH Pitfall 1 + Pitfall 4 — the fallback must never swallow `/api`).

**New line (D-02), per RESEARCH Code Examples:**
```python
app.mount("/app", StaticFiles(directory=str(BASE_DIR / "static" / "app"), html=True), name="spa")
```

> **Critical (RESEARCH Pitfall 1):** `html=True` serves `index.html` at the mount ROOT only — `/app/login`, `/app/positions` hard-reloads return **404**, not the shell. The plan MUST add an explicit fallback (RESEARCH recommends a `StaticFiles` subclass overriding `get_response` to fall back to `index.html` on 404 — self-contained in the mount, cannot shadow `/api/v2`). Browser hard-reload of `/app/login` is a required verification step.

**Auth reuse (NO server change for global-401):** `api/deps.py:43-52` `require_user` already raises `401` (no redirect) on `/api/v2` when unauthenticated, and `api/auth.py:148-154` `/auth/me` does the same. The SPA's boot-guard and global-401 redirect consume this as-is.

### `frontend/src/lib/http.ts` (utility, request-response)

**Analog (contract-exact):** `api/auth.py` + `api/deps.py` + `api/errors.py` — the fetch wrapper is a pure consumer of these. There is no JS analog, so anchor to the exact server contract it must satisfy.

**CSRF double-submit contract the wrapper MUST echo** (`api/deps.py:38-66`):
```python
CSRF_COOKIE = "telebot_csrf"
CSRF_HEADER = "x-csrf-token"
_STATE_CHANGING_METHODS = ("POST", "PUT", "PATCH", "DELETE")
# verify_csrf_token: on POST/PUT/PATCH/DELETE, compares telebot_csrf cookie to
# X-CSRF-Token header (secrets.compare_digest); mismatch/absent → 403.
```
→ Wrapper rule: read the readable `telebot_csrf` cookie (`httponly=False`, set by `api/auth.py:80-87`), set it as `X-CSRF-Token` on mutating methods only. GET passes through untouched.

**Cookie is readable by design** (`api/auth.py:70-88`, `_issue_csrf`): `httponly=False, samesite="lax", path="/"`. The session cookie `telebot_session` is `httpOnly` (`dashboard.py:227`) and MUST NOT be read in JS — `fetch(..., { credentials: "same-origin" })` sends it automatically (D-04). Nothing in `localStorage` (SPA-03 / RESEARCH anti-pattern 1).

**Error envelope the wrapper's `HttpError.body` carries** (`api/errors.py:42-46`):
```python
# {"error": {"code": "<machine-code>", "message": "<human>", "fields"?: {...}}}
# Success bodies are BARE (the resource itself) — only failures are enveloped.
# Status→code map (api/errors.py:25-34): 401→unauthorized, 403→forbidden,
#   422→validation_error, 429→rate_limited, 503→unavailable.
```
→ Wrapper rule: throw `HttpError(status, body)` on any non-2xx so BOTH QueryCache and MutationCache `onError` fire (RESEARCH Pattern 3 + D-06).

**Always use relative `/api/v2/...` URLs** (RESEARCH anti-pattern, Pitfall 4): never absolute `http://localhost:8090` — same-origin via dev proxy + prod nginx keeps the cookie attached.

### `frontend/src/lib/queryClient.ts` (provider, event-driven — the single global 401)

**Analog (contract):** `api/auth.py:148-154` (`/auth/me` 401 branch) + `api/deps.py:43-52` (`require_user` 401) — every `/api/v2` route 401s the same way; the global handler reacts to that one signal.

**The 401 source it reacts to** (`api/deps.py:43-52`):
```python
def require_user(request: Request) -> str:
    user = request.session.get("user")
    if user:
        return user
    raise HTTPException(status_code=401, detail="Session expired")  # → envelope code "unauthorized"
```

**Pattern (D-06 / D-09, RESEARCH Pattern 4):** `QueryCache` + `MutationCache` share one `onAuthError`; on `HttpError.status === 401` → `window.location.assign("/app/login")`, but **loop-break: skip the redirect when `window.location.pathname` already starts with `/app/login`**. Default query options: `placeholderData: keepPreviousData`, `refetchIntervalInBackground: false` (D-09). `staleTime`/retry = planner's call.

### `frontend/src/auth/csrf.ts` + `frontend/src/auth/LoginView.tsx` (request-response)

**Analog (contract-exact):** `api/auth.py:91-177`.

**Cold-start seeding contract** (`api/auth.py:157-177`, `GET /auth/csrf`) — the login view MUST call this on mount BEFORE the first login or it 403s (RESEARCH Pitfall 5):
```python
@router.get("/auth/csrf")
async def csrf(request: Request) -> JSONResponse:
    token = _secrets.token_urlsafe(32)
    resp = JSONResponse({"csrf_token": token})   # body token == cookie value (identical)
    resp.set_cookie(CSRF_COOKIE, token, httponly=False, samesite="lax",
                    secure=app_settings.session_cookie_secure, path="/")
    return resp
```

**Login request contract** (`api/auth.py:91-137`): `POST /api/v2/auth/login` body `{ "password": ..., "csrf_token": <echo of telebot_csrf cookie> }` + `X-CSRF-Token` header. Responses: `200 {"user":"admin"}` (sets `telebot_session`) | `401 invalid_credentials` | `403 CSRF token invalid` (cold-start) | `429 rate_limited`.

> **Login copy contract (UI-SPEC §Copywriting):** title `Telebot`, labels `Username`/`Password`, CTA idle `Log in` → pending `Logging in…`, 401 error `Incorrect username or password.`, 500 error `Something went wrong. Please try again.`, optional session-expired banner `Your session expired — please log in again.` The login form uses an uncontrolled/local input — no form lib (react-hook-form deferred to Phase 11).

## Shared Patterns

### Authentication / CSRF (consume, do not change)
**Source:** `api/auth.py:91-177` (login/logout/me/csrf), `api/deps.py:43-66` (`require_user` 401, `verify_csrf_token` 403).
**Apply to:** `http.ts` (CSRF header echo + `credentials:"same-origin"`), `csrf.ts` (cold-start seeding), `LoginView.tsx`, `queryClient.ts` (global 401), every `queryFn`/`mutationFn`.
**Rule:** read the readable `telebot_csrf` cookie → `X-CSRF-Token` on mutations; never read `telebot_session`; nothing in `localStorage`; relative URLs only.

### Error envelope
**Source:** `api/errors.py:42-46` + `:25-34` status→code map.
**Apply to:** `http.ts` (`HttpError.body` shape), `LoginView.tsx` (branch on `error.code`), any later page.
**Rule:** failures are `{"error":{"code","message","fields?}}`; success bodies are bare; throw `HttpError(status, body)` on non-2xx.

### Server-state / form-state split (SC#5 — the structural bug-class fix)
**Source:** RESEARCH Pattern 5 + `api/meta.py:43-48` (probe target).
**Apply to:** `queryClient.ts` defaults (`keepPreviousData`, `refetchIntervalInBackground:false`), `ProbeView.tsx`, ALL later pages.
**Rule:** server state = TanStack Query only; form/UI state = local React state only — never mixed; a background refetch must never reset an open input/modal.

### Dark-by-default + brand tokens
**Source:** `templates/base.html:2` (`class="dark"`), `tailwind.config.js:16` (brand hex), UI-SPEC §Color.
**Apply to:** `index.html` (root `class="dark"`), `index.css` (`@theme` role map).
**Rule:** cyan `--primary` is a 10% accent — reserved for login CTA, active nav, focus rings, wordmark ONLY; status colors (green/red/amber) are separate from the accent.

### Router precedence safety (no nginx catch-all)
**Source:** `dashboard.py:216-234` (router before mount), `nginx/telebot.conf:36-60`.
**Apply to:** `dashboard.py` mount/fallback, deploy.
**Rule:** register `/api/v2` BEFORE the `/app` mount; the deep-link fallback catches ONLY `/app/*`. nginx needs NO `/app/` block (uvicorn serves it via existing `location /`) and NO root `try_files` catch-all (would swallow `/api`, `/stream`, legacy HTMX during parallel-run — RESEARCH Pitfall 6). Verify the Phase 8 `/api/v2/auth/login` rate-limit block is deployed (currently `nginx/telebot.conf:36-45` rate-limits the LEGACY `= /login` only — the Phase 8 `/api/v2/auth/login` block is an unverified deploy artifact; planner must add a verify-step, RESEARCH Assumption A4).
