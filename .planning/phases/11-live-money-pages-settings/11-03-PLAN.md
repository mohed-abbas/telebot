<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → close/levels/close-partial | Operator-initiated live-money mutations cross to the broker |
| background poll → open modal | A 3s refetch could clobber typed SL/TP/lots if state were shared |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-07 | Tampering/Integrity | PositionsView row Close | mitigate | UI clears the row ONLY in onSuccess via invalidateQueries — no optimistic setQueryData (SC#1); a position never renders closed while live at the broker. Manual MT5-demo SC: row clears only after server 200 |
| T-11-08 | Tampering | EditPositionDialog partial-close | mitigate | Absolute lots + request_id (via usePartialClose); no percent model (Pitfall 3); zod 0<value<volume; 409 surfaced. Backed by Phase 8 test_api_idempotency |
| T-11-09 | DoS/UX-integrity | open Edit modal + drilldown under poll | mitigate | Modal/drilldown state is local React state outside the polling subtree (SC#3) — refetch cannot clobber typed input. Manual: type SL, survive ≥2×3s polls |
| T-11-10 | Spoofing | all three mutations | mitigate | Routed through api() (X-CSRF-Token echoed); rejected 403 without it (Phase 8 test_api_csrf) |
| T-11-11 | Information disclosure | mutation error toasts | mitigate | errorMessage() typed envelope only; 409 shows a fixed operator-safe string, no internals |
</threat_model>
