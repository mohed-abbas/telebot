<!-- Exact server contracts (RESEARCH §Exact Server Contracts). Success bodies are BARE; failures use {error:{code,message,fields?}}. -->
POST /api/v2/positions/{account}/{ticket}/close        → MutationResult {ok,success,error}. No body.
POST /api/v2/positions/{account}/{ticket}/levels       → {ok,success,changed:{sl?,tp?},error}. Body CloseLevelsIn {sl?:float, tp?:float} (both optional; null=keep). No-op → {ok:true,changed:{}}. 422 if sl/tp ≤ 0.
POST /api/v2/positions/{account}/{ticket}/close-partial → {ok,success,closed_volume,closed_volume_display,error}. Body PartialCloseIn {close_volume:float, request_id:str}. 409 = id reused w/ different params OR in-flight retry. 422 = close_volume out of range. Same id+same params → cached 200 replay (broker untouched).
POST /api/v2/emergency/close                            → EmergencyResult {results,ok}. No body.
POST /api/v2/emergency/resume                           → {status:"resumed"}. No body.
POST /api/v2/settings/{account}/validate               → SettingsValidateResult {valid, errors:{field:msg}, diff:[{field,old,new}], dry_run_text}. Body {account, values:{…}}. RETURNS 200 EVEN WHEN valid:false.
POST /api/v2/settings/{account}                         → MutationResult {ok,success}. Body {account, values:{…}}. Re-validates server-side; 422 if caps re-breached.
POST /api/v2/settings/{account}/revert                 → MutationResult {ok,success}. Body {account}. Inverts ONLY the latest change (no audit_id). 404 "Nothing to revert".

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/v2 mutations | Untrusted client requests cross into live-money + settings operations |
| client retry → partial-close | A retried close request could double-fire and close the wrong volume |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-03 | Spoofing/Tampering | every mutation hook | mitigate | All hooks call api() which echoes X-CSRF-Token (double-submit) — server rejects 403 without it; never raw fetch (Pitfall 2). Asserted by grep (no `fetch(`) + Phase 8 test_api_csrf |
| T-11-04 | Tampering | usePartialClose double-fire | mitigate | Absolute close_volume + stable request_id reused on pure retries → server idempotency replays cached 200 (no second broker hit); 409 on id-reuse-diff-params surfaced to operator (Pitfall 3) |
| T-11-05 | Repudiation/Integrity | useClose/useLevels/usePartialClose | mitigate | No optimistic setQueryData (SC#1) — a position can never render closed while live at the broker; UI state derives only from server-confirmed invalidateQueries |
| T-11-06 | Information disclosure | mutation onError toasts | mitigate | Toast copy comes only from errorMessage() (the typed {error:{message}} envelope) — no raw exception/stack leaked to the viewport |
</threat_model>
