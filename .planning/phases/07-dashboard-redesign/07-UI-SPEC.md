## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form field gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (rare) |
| 3xl | 64px | Page-level spacing (reserved for full-page layouts) |

Exceptions:
- Touch targets on mobile: minimum 44px height for tappable elements (hamburger icon, nav links in drawer)
- Table cells: 12px vertical padding (py-3) for data density

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 12px (text-xs) | 400 (normal) | 1.4 |
| Heading | 18px (text-lg) | 600 (semibold) | 1.2 |
| Display | 24px (text-2xl) | 600 (semibold) | 1.2 |

**Rationale:** Two-weight system maximizes visual consistency while maintaining clear hierarchy. 400 (normal) for all body and label text; 600 (semibold) for all headings and display text. 14px body maintains data density for trading dashboards. 12px labels for table headers and form field labels.

## Interaction Patterns

### Mobile Navigation (DASH-02)
- Hamburger icon visible below `md` breakpoint (768px)
- Click triggers sidebar `aria-hidden="false"` state
- Backdrop overlay (bg-black/50) dismisses on click
- Sticky header shows page title + hamburger

### Positions Drilldown (DASH-03)
- Click position row toggles `<details>` open state
- Multiple rows can be expanded simultaneously (D-06)
- Expanded panel shows: fill history table, per-stage SL/TP, current P/L (SSE live), signal attribution link
- P/L updates via existing SSE stream (2s cadence)

### Analytics Time Filter (DASH-04)
- Horizontal pill tabs: `7d | 30d | 90d | All`
- Single active state; click switches immediately
- HTMX-driven: `hx-get="/analytics?range=7d"` swaps content

### Trade History Filters (DASH-05)
- Inline filter bar: Account dropdown, Source dropdown, Symbol dropdown, Date range (from/to)
- Filter combination is AND logic
- URL params for persistence: `?account=X&symbol=Y&from=2026-04-01`
- HTMX partial swap on filter change

### Settings Toasts (SEED-001)
- Success toast: "Settings saved" with checkmark icon, auto-dismiss 4s
- Error toast: "{field}: {error message}" with X icon, manual dismiss
- Revert confirmation toast: "Reverted {field} to {value}" with undo context
- Position: bottom-right (Basecoat default)

### Field Labels (Settings UX Polish D-15)

| DB Column | User-Facing Label | Help Text |
|-----------|------------------|-----------|
| risk_mode | Risk calculation | How lot size is determined: "Percent of balance" calculates from equity; "Fixed lot" uses exact lot size. |
| risk_value | Per-trade risk | When percent: 0.5-3% is typical. When fixed: the exact lot size (e.g., 0.04). |
| max_stages | Maximum entries per signal | How many positions to open for a staged signal. Higher = more exposure. Range: 1-10. |
| default_sl_pips | Default stop-loss (pips) | Stop-loss for text-only signals before zone info arrives. Range: 10-500 pips. |
| max_daily_trades | Daily trade limit | Maximum new signals processed per day. Resets at midnight UTC. Range: 1-100. |

## Loading States

### Skeleton Patterns
- Tables: Pulse-animated rows with grey rectangles matching column widths
- Cards: Full card skeleton with header and body placeholders
- Position drilldown: Collapsed skeleton inside expanded `<details>` until data loads

### SSE Reconnection
- Existing pattern acceptable: 2s cadence, automatic reconnection
- Visual: "Live" indicator in header pulses when reconnecting
- No user action required; silent recovery

## Phase-Specific Constraints

1. **Incremental restyle** (D-16): One page at a time. Compat shim stays until all pages converted.
2. **Restyle order** (D-17): base.html (mobile nav) -> overview -> positions -> history -> analytics -> settings -> signals -> staged
3. **No new dependencies**: No new CSS or JS libraries beyond vendored Basecoat
4. **Zero regressions**: Every existing view must remain functional throughout restyle
5. **HTMX patterns preserved**: All existing `hx-*` attributes, SSE streams, and partial swaps continue working
