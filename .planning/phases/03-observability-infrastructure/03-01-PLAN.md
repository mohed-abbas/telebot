       if keyword_count >= 1 and has_price:

       return False

2. Create `docs/server-messages.md` with the following content documenting OBS-02:

   Title: "Server Message Limits"

   Sections:
   - **What is a "server message"?** -- Explain this is a bot-internal safety limit (configured via `max_daily_server_messages` in GlobalConfig, default 500). Each MT5 API call that sends a request to the broker's trade server counts as a server message. This includes: `open_order()`, `modify_position()`, `close_position()`, `cancel_order()`. It does NOT include: `get_positions()`, `get_account_info()`, `get_price()` (these are read-only queries).

   - **How is it tracked?** -- The `daily_stats` table tracks `server_messages` per account per day. The `increment_daily_stat(account_name, "server_messages")` function is called after each write operation to MT5. The counter resets daily (UTC date boundary via `_utc_today()`).

   - **MT5 broker limits** -- MT5 brokers may impose their own server message limits (varies by broker). These are separate from the bot's internal limit. Check with your broker for their specific limits. The bot's limit is a conservative safety net to prevent runaway order placement.

   - **Configuration** -- Set via `accounts.json` under `global.max_daily_server_messages`. Default: 500. This is a per-account limit. When the limit is reached, the bot will skip new trade executions for that account until the next UTC day.

   - **Monitoring** -- The dashboard overview page shows the current `daily_messages` count for each account. No percentage warning is currently implemented for server messages (only for trade count via EXEC-04).
  </action>
