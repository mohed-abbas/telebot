<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/curl → /api/v2 | Untrusted HTTP crosses into the FastAPI app; session cookie + CSRF token carried here |
| api/ → db._pool | New package reads the existing pool as an accessor; must never mutate bot-core DDL |
| api/ → bot core (executor/connector) | New code calls live objects; must never import-bind or edit them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing/Tampering | verify_csrf_token on /api/v2 mutations | mitigate | Double-submit `telebot_csrf` cookie + `X-CSRF-Token` header compared with `secrets.compare_digest`; 403 on mismatch (D-15). Tested in Plan 02. |
| T-08-02 | Information Disclosure | api/errors.py handler | mitigate | Enveloped `{error:{code,message}}` only — no raw `detail`/traceback/stack leakage (V13). |
| T-08-03 | Tampering | idempotency_keys DDL placement | mitigate | DDL lives in api/idempotency.py via `db._pool` accessor; db.py byte-for-byte untouched (diff guard, Task 3). |
| T-08-04 | Elevation/Spoofing | require_user dependency | mitigate | Session-read 401 branch reused on every authed /api/v2 route (V4 access control). |
| T-08-05 | Information Disclosure | telebot_csrf cookie attrs | accept | Cookie is `httponly=false` (SPA must read it) but grants nothing alone — not a session credential; session stays httpOnly. |
| T-08-SC | Tampering | npm/pip installs | accept | Zero new packages this phase (RESEARCH Package Legitimacy Audit — all deps already pinned in requirements.txt/.venv). No install task; no checkpoint required. |
