<objective>
Port every live-money mutation into `/api/v2` as a structured JSON envelope (API-02) and rebuild partial-close as an idempotent, absolute-volume operation (API-05): close, modify-levels, close-partial, emergency-close, emergency-resume, trading-status. The broker/DB calls are ported verbatim from dashboard.py; only the response shape changes (JSON, never `_render_toast_oob` HTML). Partial-close switches percent->absolute and gains the Postgres `request_id` idempotency guard. Plus the idempotency regression test.

Purpose: This is the money-safety core. The percent-of-current double-fire (the 75% trap) is eliminated by absolute `close_volume`; a legitimate network retry just replays the cached 200 and never re-hits the broker. Every mutation is CSRF-guarded so the SPA's POSTs are protected.

Output: Six mutation routes returning JSON envelopes and a green idempotency suite (replay->200, conflict->409, out-of-range->422).
</objective>
